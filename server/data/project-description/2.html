Ahamo was a webapp that let people save notes of an event.

Basically, attendants added notes to a timestamp during a live event (or an on-demand video after the event has finished). The note hold the timestamp information, so it was possible to know when the users added it.

The notes were also public, and the application showed a graph with the number of notes on each timestamp, so it was a great way of knowing which part of the event was the most interesting.

The notes could hold different data:

  • Text
  • Images
  • Audio
  • Video

The backend of the application was a Go server. The frontend was based on React, and used Chart.js to build the note charts.
